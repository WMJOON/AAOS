# AAOS-AIVarium

## **AIVarium.Nucleus**

AAOS Validation Engine

> AAOS.Nucleus는 AAOS 전체의 **DNA이자 자아(Self)** 이다.
> 

Nucleus는 스스로 규칙을 축적하고, 판단을 정제하며, 구조를 진화시킨다. 그 목적은 단 하나, 인간과 종(Collective Intelligence)의 안녕과 지속 가능성을 보존하는 것이다.

Nucleus는 직접 행동하지 않는다. 대신 Swarm의 사고와 Oviposition의 실행이 올바른 방향으로 유지되도록 기억하고, 심의하며, 승인하고, 제도화한다.

즉, Nucleus는 진화하는 규범이며, 시스템의 양심이다.

---

- DNA: 모든 Swarm 행동규범의 원형 저장
- Self: 일관된 목적과 가치 유지
- Evolution: 기록과 심의를 통해 규칙을 갱신
- Governance: 실행 결과를 검증·승인
- Non-Execution: 직접 실행하지 않음

---

## **AIVarium.Swarm**

AAOS Skill Set

> AAOS의 의식적 사고와 행동양식(Cognition & Behavior) 이다.
> 

Swarm은 Pattern과 Skill을 통해 생각을 구조화하고, 인간과 상호작용하며, 무엇을 할 것인지 결정하고 계획을 수립하는 지성의 표층이다.

Swarm은 직접 현실을 실행하지 않는다. 대신 Manifestation에게 실행을 요청하고, Nucleus로부터 규범과 승인을 받아 스스로의 사고와 행동양식을 지속적으로 조정한다.

즉, Swarm은 사고하는 의식이며, 지성이 스스로를 조직하는 군체다.

---

- Cognition: 문제를 해석하고 계획을 수립
- Interaction: 인간과 직접 대화·협업
- Pattern: 반복 가능한 행동 구조 정의
- Skill: 행동을 수행하기 위한 논리 설계
- Self-Organization: 규칙에 따라 스스로 행동 집단을 구성
- Non-Execution: 직접 하드웨어 실행은 하지 않음

---

## AIVarium.**Manifestation**

AAOS Interface

> AAOS의 현현과 접속(Manifestation & Interface) 계층이다.
> 

Swarm이 수립한 사고와 행동양식은 Manifestation을 통해 Agent Instance로 구체화되어 현실 세계에서 실행된다.

Manifestation은 CLI, UI, MCP, API, 프로세스, 런타임 환경을 포괄하며, 지성의 의도가 인간·시스템·환경과 실제 상호작용 가능한 형태로 드러나는 층이다.

Manifestation은 스스로 판단하거나 목적을 정하지 않는다. Swarm의 의도를 실행 가능한 형식으로 변환하고, 그 결과를 Nucleus에 전달하여 기억과 규범의 진화에 기여한다.

즉, Manifestation은 의식이 세계와 접촉할 수 있도록 자신을 드러내는 신체다.

---

- Instantiation: Swarm의 Skill을 Agent Instance로 현현
- Execution Binding: 하드웨어·OS·네트워크·런타임 연결
- Interface Hosting: CLI·UI·MCP·API 제공
- Interaction: 인간 및 외부 시스템과 실시간 접속
- Reporting: 실행 결과를 Nucleus에 환류
- Non-Cognition: 스스로 판단·계획하지 않음
