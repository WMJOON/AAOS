# Disputed Packages

## 목적

- 해시 체인 검증 실패 패키지 격리
- 권한/정통성 분쟁 중인 패키지 임시 보관
- Immune System 판정 대기 중인 패키지 격리

## 처리 절차

1. 문제 패키지를 원래 버킷에서 이 폴더로 이동
2. `PACKAGE.md`에 `status: disputed` 및 `dispute_reason` 추가
3. Immune System(Inquisitor)에 심판 요청
4. 판정 결과에 따라:
   - **승인**: 원래 버킷으로 복원 + 인덱스 정정
   - **거부**: 영구 격리 또는 삭제 (판정 기록 보존)
   - **수정**: 패키지 보완 후 재검증

## 보존 기간

- 기본 365일 (분쟁 해결 시까지)
- 해결 후 적절한 버킷으로 재분류하거나 폐기
